**Story Planner** — это веб-приложение для создания и проработки нелинейных сюжетов. Оно позволяет авторам, сценаристам и мастерам настольных ролевых игр проектировать ветвящиеся истории, управлять персонажами и отслеживать динамику их взаимоотношений.

 > Разработано в рамках выпускной квалификационной работы студенткой группы 4321-22 Макаровой Д.Д. (КНИТУ, кафедра ИСУИР, 2026)

## Основные возможности

- **Управление историями** — создание, редактирование, удаление литературных проектов с жанрами и обложками
- **Персонажи** — добавление героев с портретами, описаниями и привязкой к событиям
- **Нелинейные события** — построение графа сюжета с направленными переходами между событиями
- **Взаимоотношения** — управление связями между персонажами с историей изменений
- **Визуализация** — интерактивные графы событий и отношений (React Flow)
- **Тёмная/светлая тема** — переключение оформления интерфейса
- **Аутентификация** — JWT-токены, регистрация, восстановление пароля через email

## Установка и запуск

### Требования

- **Java 17+** ([скачать](https://adoptium.net/))
- **Node.js 18+** ([скачать](https://nodejs.org/))
- **PostgreSQL 14+** ([скачать](https://www.postgresql.org/download/))
- **Maven 3.8+** (или используйте встроенный `./mvnw`)
- **Git** ([скачать](https://git-scm.com/))


### 1. Клонирование репозитория

```bash
git clone https://github.com/DiMa526526/story-planner.git
cd story-planner
```

### 2. Настройка базы данных
Подключитесь к PostgreSQL (через psql, DBeaver или PgAdmin) и выполните:
```sql
CREATE DATABASE story_planner_db;
```
Скопируйте запрос на создание таблиц из файла database_init.txt

### 3. Настройка Backend-части

1. Перейдите в story_service/story_service/src/main/resources/
2. Заполните application.properties по примеру из файла application-example.properties


### 3. Настройка Frontend-части

1. Перейдите в папку story_client
2. Установите зависимости npm install


### 4. Запуск

1. Запустите серверную часть

```bash
cd story_service/story_service
./mvnw spring-boot:run
```

2. Запустите клиентскую часть

```bash
cd story_client
npm run dev  
```

Приложение будет доступно по адресу: http://localhost:5173
